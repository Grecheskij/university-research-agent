# University Research Agent

## Назначение проекта

Проект предназначен для поддержки научных исследований в университете. Основные пользователи: преподаватели и докторанты. Развёртывание планируется на Hugging Face Spaces. Агент и интерфейс обязаны полноценно поддерживать русский и английский языки.

## Общие правила

- Стек проекта фиксирован и не может изменяться без явного указания пользователя в этом файле.
- Основной LLM: Google Gemini через `google-generativeai`, модель `gemini-2.0-flash`.
- Резервный LLM: Groq API, модель `llama-3.3-70b-versatile`.
- Агентная логика и цепочки: `LangChain >= 0.3` и `langchain-google-genai`.
- Векторное хранилище: локальный персистентный `ChromaDB`.
- API слой: `FastAPI`, `Uvicorn`, `Pydantic v2`.
- Интерфейс: `Gradio >= 4.0`.
- HTTP клиент: только `httpx` в асинхронном режиме.
- Переменные окружения: только через `.env` локально и секреты платформ в продакшене.
- Retry-логика: `tenacity`.
- Python: `3.11+`.
- Типизация обязательна во всём проекте. `Any` использовать только при обоснованной необходимости.
- Все публичные функции должны иметь docstring.

## Локализация и язык ответов

- UI обязан поддерживать переключение языка `RU/EN` для всех пользовательских текстов.
- Агент принимает запросы и возвращает ответы на русском и английском языках.
- Язык запроса определяется автоматически.
- По умолчанию ответ формируется на языке запроса, если пользователь явно не запросил другой язык.

## Инженерные требования

- Все внешние HTTP-вызовы выполняются через `async httpx` с `timeout=30`.
- Все async-функции используют `httpx`, а не `requests`.
- Для интеграций с внешними API обязательны `try/except` и retry через `tenacity`.
- Нужно логировать запросы к внешним API: endpoint, источник, статус, время выполнения.
- Нельзя логировать секреты и полные тела пользовательских запросов.
- Все ключи, токены и чувствительные параметры запрещено хардкодить в коде или репозитории.
- Валидация и внутренние модели данных должны быть построены на `Pydantic v2`.

## Безопасность

- CORS должен быть минимальным: только localhost для разработки и домен Hugging Face Space для продакшена.
- `/docs` и `/redoc` в продакшене должны быть скрыты или защищены, если публичный доступ не нужен.
- Чувствительные маршруты должны защищаться аутентификацией, если такие маршруты появятся.
- Размер тела запроса и частота запросов должны ограничиваться на уровне приложения или прокси.
- Все секреты должны храниться только в `.env`, Hugging Face Secrets и GitHub Secrets.

## Git и GitHub

- Проект ведётся в git-репозитории.
- Основная ветка: `main`.
- Новые значимые изменения выполняются в отдельных ветках вида `feature/...` с последующим PR в `main`, если пользователь не дал иное указание.
- Значимые шаги фиксируются небольшими осмысленными коммитами.
- Сообщения коммитов оформляются в стиле `feat: ...`, `fix: ...`, `refactor: ...`, `test: ...`, `chore: ...`.
- Репозиторий должен быть опубликован на GitHub.
- Ссылка на GitHub-репозиторий указывается в `devops/README.md`.
- GitHub Actions допускаются как опциональное улучшение, если пользователь попросит.

## Интеграция skills GitHub и Hugging Face

В среде разработки доступны GitHub- и Hugging Face-инструменты. Они используются как вспомогательные средства и не должны нарушать архитектуру проекта.

### GitHub / CI

- `GitHub`: использовать для создания и настройки репозитория `university-research-agent`, просмотра PR, issues, веток и статусов CI.
- `Publish Changes`: использовать для осмысленных коммитов, push и публикации изменений.
- `Review Follow-up`: использовать для обработки замечаний ревью и корректировок в PR.
- `CI Debug`: использовать для анализа и устранения проблем GitHub Actions.

Все GitHub/CI skills обязаны соблюдать git-процесс этого файла: `main`, feature-ветки, маленькие осмысленные коммиты и предсказуемая история.

### Hugging Face

- `Hugging Face: hf CLI`: использовать для создания и настройки Hub-репозитория и Space, синхронизации кода и управления секретами.
- `Hugging Face: huggingface Gradio`: использовать для разработки и рефакторинга Gradio UI.
- `Hugging Face: huggingface Datasets`: использовать только как вспомогательный источник примеров и тестовых данных.
- `Hugging Face: huggingface Jobs`, `LLM Trainer`, `Vision Trainer`, `Trackio`: использовать только по явной инструкции пользователя.
- `Hugging Face: huggingface Papers` и `Paper Publisher`: использовать опционально, если понадобится анализ или публикация научных материалов.

### Прочие системные skills

- Вспомогательные skills вроде `Skill Creator`, `Skill Installer`, `Plugin Creator`, `OpenAI Docs` и `Image Gen` можно использовать только для поддержки процесса разработки.
- Запрещено использовать вспомогательные skills для замены стека или изменения архитектурных ограничений проекта.
